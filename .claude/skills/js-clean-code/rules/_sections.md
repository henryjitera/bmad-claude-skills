# Section Definitions

This file defines the sections and their metadata for organizing clean code rules.

## Section 1: Variables
- **Prefix:** `var-`
- **Impact:** HIGH
- **Description:** Best practices for naming and declaring variables to improve code readability and maintainability.

## Section 2: Functions
- **Prefix:** `func-`
- **Impact:** CRITICAL
- **Description:** Core principles for writing clean, single-purpose functions that are easy to test and understand.

## Section 3: Objects and Data Structures
- **Prefix:** `obj-`
- **Impact:** MEDIUM-HIGH
- **Description:** Patterns for structuring objects and managing data encapsulation.

## Section 4: Classes
- **Prefix:** `class-`
- **Impact:** MEDIUM-HIGH
- **Description:** Guidelines for designing clean, maintainable ES6+ classes.

## Section 5: SOLID Principles
- **Prefix:** `solid-`
- **Impact:** HIGH
- **Description:** The five SOLID principles adapted for JavaScript development.

## Section 6: Testing
- **Prefix:** `test-`
- **Impact:** CRITICAL
- **Description:** Best practices for writing clean, maintainable tests.

## Section 7: Concurrency
- **Prefix:** `async-`
- **Impact:** MEDIUM
- **Description:** Guidelines for handling asynchronous code with promises and async/await.

## Section 8: Error Handling
- **Prefix:** `error-`
- **Impact:** HIGH
- **Description:** Best practices for catching and handling errors properly.

## Section 9: Formatting
- **Prefix:** `format-`
- **Impact:** MEDIUM
- **Description:** Code formatting conventions for consistency and readability.

## Section 10: Comments
- **Prefix:** `comment-`
- **Impact:** LOW-MEDIUM
- **Description:** Guidelines for when and how to write useful comments.
